Making Interfaces Adaptive

Preliminary Investigations

Communicating Adaptions

Mobile phone usage patterns over time can provide a wealth of data about user habits and preferences, but the availability of this data has had relatively little impact on the design of mobile user interfaces that improve the user experience by adapting to individual users.

The goal of this project is to identify design opportunities in every day apps where the UI can adapt to user habits to provide a more seamless interaction.

Mobile app users can sometimes have repetitive patterns and habits in the way they use the app. Consider, for example, the customer who always uses their Starbucks app for payment when in a Starbucks store. It is inefficient for this customer to land on the general Starbucks page and spend extra time maneuvering to the payments screen every time. Instead, a responsive UI will learn that:

"when this user is in a Starbucks store" -> "the user will need the payments screen".

This kind of repeated usage data is an unexplored source of information that can be used to create adaptive UIs that are customized to each user's unique habits, predicting user needs and thus reducing unnecessary steps.

I first made some preliminary observations of the mobile apps people around me used frequently. In particular, I took note of situations where certain interactions with the UI could be predicted.

One of the cases that I found striking, and which I chose to focus on, was the Forever 21 mobile shopping app. Users had to filter for their preferences each time they browsed a new category of items. None of these preferences were saved when users navigated away from a category, which means they sometimes had to input their sizes and color selections multiple times within the same session.

Constant selection and re-selection of preferences is exactly the kind of repetitive interaction that adds little to the user experience. Since an individual's preferences, such as size and price range, are unlikely to differ much within the same browsing session, I felt that a shopping app can easily adapt and filter for these preferences automatically.

The challenge is to communicate to the user that such an adaptation has taken place. Below are some initial mock-ups I created. The main idea was for the loading screen animation to indicate that user preferences are being filtered for.

The feedback I received was that the loading screen provided too little information about the preferences being filtered for. I created a new animation that more closely resembled the existing preferences selection interface.

Although the above loading animation design was informative about the preferences being selected for, it was time consuming and could be tiresome if viewed multiple times. I thus challenged myself to fit the entire loading animation in under 2 seconds.
